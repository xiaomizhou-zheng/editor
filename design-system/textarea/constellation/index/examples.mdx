## Validation

This behavior is how text area will behave within [forms](/patterns/forms).

Validation displays an error message related to the restrictions of the text area.

When a user selects the text area and starts typing or changing content, the focus color changes to blue. When validating text areas in real-time, message icons switch based on the message type. For example, helper text becomes an error message when the input content doesn't meet the criteria. Error and warning messages disappear when the criteria is met.

Keep helper text as short as possible. For complex information, provide a link to more information in a new browser tab. Use the [messaging guidelines](/content/messaging-guidelines)Â for more help.

## Resize

### Smart

A text area that shows all user input at once. Overflow text wraps onto a new line and expands the text area.

### Auto

A text area that lets the user resize the area horizontally and vertically.

### Vertical / horizontal resize

A text area that lets the user resize the area either vertically only or horizontally only.

### None

A text area that does not resize and uses a scroll bar to show overflow text.
