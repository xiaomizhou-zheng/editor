<Example Component={DatePickerForm} packageName="@atlaskit/datetime-picker" />

### Validation

This is how date picker behaves within [forms](https://atlaskit.atlassian.com/packages/design-system/form).

Validation displays an error message related to the restrictions of the date picker.

When a user selects the date picker area, the focus color changes to blue. When validating date pickers in real-time, message icons switch based on the message type. 

For example, helper text becomes an error message when the input content doesn't meet the criteria. Error and warning messages disappear when the criteria is met.

Keep helper text as short as possible. For complex information, provide a link to more information in a new browser tab. Use the [messaging guidelines](/content/messaging-guidelines) for more help.

## Disabled dates

If a certain date is not a valid selection, you may disable it in the calendar shown to users. This does not restrict the dates that a user may type, so validation is necessary. 

### Specific dates

Use `disabled` to disable individual dates.

<Example Component={DatePickerDisable} packageName="@atlaskit/datetime-picker" /> 

### Date ranges

Use `minDate` to set a minimum valid date and `maxDate` to set a maximum valid date. These can be used to define a valid date range.

### Complex behavior

Use `disabledDateFilter` for more complicated situations.

## Internationalization

`DatePicker` supports internationalization through two props:

- `locale` affects language, format and, parsing.
- `weekStartDay` determines the first day of the week shown on the calendar.

### Locale

Use `locale` to tailor UI copy to local audiences.

<Example Component={DatePickerLocale} packageName="@atlaskit/datetime-picker" />

### Week start day

Use `weekStartDay` to adjust which day of the week is shown first in the calendar. A value of `0` corresponds to Sunday (default), `1` to Monday, and so on.

<Example Component={DatePickerWeekStartDay} packageName="@atlaskit/datetime-picker" />

## Date formats

You can customize the date format using the `dateFormat` prop. Formats are given as strings and use the syntax specified at [Modern JavaScript date utility library](https://date-fns.org/).
